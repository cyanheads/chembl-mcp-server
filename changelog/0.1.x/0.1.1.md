# 0.1.1 — 2026-06-25

First release published to npm (`v0.1.0` was an unpublished scaffold tag). Drug-discovery data over [ChEMBL](https://www.ebi.ac.uk/chembl/) (EBI) — the curated compound ↔ target ↔ bioactivity link, plus drug mechanisms and clinical indications. Keyless upstream; CC BY-SA 3.0 attribution carried in server-level instructions.

## Added

- **ChEMBL surface** — `chembl_search_molecules` (name / ChEMBL ID / InChIKey lookup or `exact` | `similarity` | `substructure` structure search from a SMILES), `chembl_get_bioactivities` (the flagship compound↔target bridge; large sets spill to a DataCanvas `bioactivities` table), `chembl_search_targets`, `chembl_get_drug_info`, and `chembl_get_assay`. Eight tools total — seven always registered, plus `chembl_dataframe_drop` opt-in behind `CHEMBL_DATAFRAME_DROP_ENABLED`.
- **`chembl_dataframe_query`** / **`chembl_dataframe_describe`** — read-only SQL over the spilled bioactivity rows; return `canvas_disabled` unless `CANVAS_PROVIDER_TYPE=duckdb`.
- **Resources** `chembl://molecule/{chemblId}` and `chembl://target/{chemblId}` — injectable-context mirrors of the per-record fetch.
- **`ChemblService`** — single upstream client coercing ChEMBL string numerics to `number | null` (absent → `null`, never `0`). All eight tools and both resources route through one `fetchJson` chokepoint that sanitizes upstream HTTP errors, so framework status codes, response bodies, and internal URLs never reach the client.

## Changed

- **npm scope** — published as `@cyanheads/chembl-mcp-server` (was the bare `chembl-mcp-server`). The README `<h1>`, npm badge, Cursor + VS Code install payloads, `bunx` / `npx` args, both plugin manifests, and `server.json` `packages[].identifier` mirror the scoped name; `runtimeHint` `node` → `bun`.
- **`createApp` identity** — dropped `websiteUrl`; the `initialize` / well-known identity block is now `name` + `title` only (`description` derives from `package.json`).
- **Package metadata** — `package.json` gains `mcpName`, `bugs`, `homepage`, `funding`, `author`, `packageManager`, and expanded keywords; `manifest.json` gains `repository` / `homepage` / `license`; `server.json` documents all six `CHEMBL_*` / `CANVAS_PROVIDER_TYPE` environment variables.
- **Description** — rewritten to a benefit-led one-liner across the npm, registry, Docker, and plugin surfaces (≤100-char short form where the MCP Registry caps it).
- **Dependencies** — `@types/node` (dev) `^26.0.0` → `^26.0.1`.
