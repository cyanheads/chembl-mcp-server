# 0.1.0 — 2026-06-23

Drug-discovery data over [ChEMBL](https://www.ebi.ac.uk/chembl/) (EBI) — the curated link between compounds, protein targets, and measured bioactivity, plus drug mechanisms and clinical indications. Keyless upstream; CC BY-SA 3.0 attribution carried in server-level instructions.

## Added

- **`chembl_search_molecules`** — discovery entry point. Name / ChEMBL ID / InChIKey lookup (`search_type=name`) or structure search (`exact` | `similarity` | `substructure`) from a SMILES, routed by a `search_type` enum to the matching ChEMBL endpoint. `max_phase` on every row; Tanimoto `similarity` percent on structure searches.
- **`chembl_get_bioactivities`** — the flagship compound↔target bridge. Bioactivity measurements for a molecule **or** a target (handler-level XOR gate), filterable by `standard_type` / `pchembl_value_min` / `assay_type` / `organism`, ranked on `pchembl_value`. Large sets spill to a DataCanvas `bioactivities` table; inline preview capped to `limit`.
- **`chembl_search_targets`** — resolve a UniProt accession / gene symbol / name to the ChEMBL target ID, flattening component accessions + `GENE_SYMBOL*` synonyms per component.
- **`chembl_get_drug_info`** — drug pharmacology (mechanisms, targets, action type, first-approval, indications) composed via `Promise.allSettled` so a missing list degrades to empty.
- **`chembl_get_assay`** — assay provenance (type, target, organism, 1–9 confidence score) behind a bioactivity row.
- **`chembl_dataframe_query`** / **`chembl_dataframe_describe`** — read-only SQL over the spilled `bioactivities` table, and table/column discovery. The standardized DataCanvas consumer surface; `canvas_disabled` error when `CANVAS_PROVIDER_TYPE` is not `duckdb`.
- **`chembl_dataframe_drop`** — opt-in (`CHEMBL_DATAFRAME_DROP_ENABLED=true`), conditionally registered so it is absent from `tools/list` when off.
- **Resources** `chembl://molecule/{chemblId}` and `chembl://target/{chemblId}` — injectable-context mirrors of the per-record fetch, `CHEMBL\d+`-validated.
- **`ChemblService`** — single upstream client: Django-style filtered URL builder, `page_meta` pagination, `withRetry`-wrapped fetch+parse, string→`number | null` numeric coercion (absent → `null`, never `0`), nested-structure flattening, and an async-iterable activity page stream feeding `spillover()`.
- **Config** — `CHEMBL_API_BASE_URL`, `CHEMBL_REQUEST_TIMEOUT_MS`, `CHEMBL_MAX_PAGE_SIZE`, `CHEMBL_DEFAULT_LIMIT`, `CHEMBL_DATAFRAME_DROP_ENABLED`, plus `CANVAS_PROVIDER_TYPE` gating the analytical SQL path. `@duckdb/node-api` added for the DataCanvas spill.
