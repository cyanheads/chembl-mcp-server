# 0.2.0 — 2026-06-27

Breaking output change on the flagship: `chembl_get_bioactivities` renames its `totalFound` field to `totalCount`. The rest of the release tightens bioactivity ranking and numeric fidelity.

## Changed

- **`chembl_get_bioactivities` output — `totalFound` → `totalCount`** (breaking) ([#1](https://github.com/cyanheads/chembl-mcp-server/issues/1)). Same value (the full upstream match count before any preview cap), standardized on the `totalCount` name the sibling search tools already use. The count now lives in the output schema and is rendered by `format()`; it was previously mirrored only in the enrichment trailer. Callers reading `result.totalFound` must switch to `result.totalCount`.
- **`canvasDisabled` promoted into the `chembl_get_bioactivities` output schema** ([#1](https://github.com/cyanheads/chembl-mcp-server/issues/1), [#4](https://github.com/cyanheads/chembl-mcp-server/issues/4)). Was an optional enrichment field; now a required output field so the `content[]` renderer can branch on it.
- **`chembl_get_bioactivities` default preview is potency-ranked** ([#3](https://github.com/cyanheads/chembl-mcp-server/issues/3)). The preview/spill stream now sends `pchembl_value__isnull=false`, so measurements without a derivable `pchembl_value` — which ChEMBL sorts first under the descending `-pchembl_value` order — no longer dominate the default view. `totalCount` stays the honest full match count via a new `countActivities` request that omits the filter; the agent-facing notice discloses the potency-filtered subset when it narrows the set.

## Fixed

- **`chembl_dataframe_query` BIGINT aggregates returned as strings** ([#2](https://github.com/cyanheads/chembl-mcp-server/issues/2)). `COUNT(*)`, `SUM(...)` over integer columns, and the native `activity_id` came back as JSON strings (DuckDB-Node's default to avoid 53-bit precision loss); they are now coerced to numbers when the value is a canonical integer within `Number.isSafeInteger`. VARCHAR columns (e.g. the raw `value` passthrough) are protected via the staged-table column types, and out-of-safe-range integers stay strings to preserve precision.
- **`chembl_get_bioactivities` canvas-disabled preview text** ([#4](https://github.com/cyanheads/chembl-mcp-server/issues/4)). The `content[]` header read `spilled: no (preview is the full set)` even when `CANVAS_PROVIDER_TYPE` was not `duckdb` and the inline rows were capped at `limit`; it now states the canvas is disabled and the rows are a capped preview, not the complete set.
